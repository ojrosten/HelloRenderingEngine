`dependencies/sequoia/Source/sequoia/Maths` + `Physics` (as of 2026-07). Calibration from the user: **maths/spaces and Physics are in very considerable flux — do not rely on fine details there**; Graph is mature and load-bearing.

**Maths/Graph** (stable, go deep when needed):
- Flavours: `graph_flavour{undirected, undirected_embedded, directed}` (no directed-embedded). "Embedded" = each edge records its `complementary_index` in the target node's edge list — a combinatorial embedding preserving edge ordering around nodes.
- Families: dynamic `directed_graph`/`undirected_graph`/`embedded_graph` (`DynamicGraph.hpp`; templated on EdgeWeight/NodeWeight/EdgeMetaData/storage configs; tree-shaped construction via `tree_initializer`); constexpr-capable `static_*_graph` (`StaticGraph.hpp`, smallest-sufficient index type); `heterogeneous_*_graph` (per-node weight types in a tuple); trees in `DynamicTree.hpp` (`tree_base`, `tree_adaptor`). Traversal layer is written against concepts in `GraphTraits.hpp` (`network`, `dynamic_network`, ...).
- Storage: connectivity is a partitioned sequence of edges (one partition per node; `Connectivity.hpp`, the 1700-line workhorse) over `bucketed_edge_storage_config` (default) or `contiguous_edge_storage_config` from Core's PartitionedData. Empty-weight nodes are entirely implicit. Undirected edges are stored twice; edge-weight sharing between the partner half-edges is automatic for big/non-copyable weights via `object::shared` vs `object::by_value` handlers (overridable via `edge_sharing_preference`). Allocator-aware throughout — this is what the allocation-testing machinery exercises hardest.
- Traversals (`GraphTraversalFunctions.hpp`): one overload set `traverse(breadth_first|depth_first|pseudo_depth_first|priority_first, graph, traversal_conditions, ...visitors)`; visitor slots for node-before/after-edges and per-edge; `find_disconnected_t`/`ignore_disconnected_t` control component restarts. Concurrency is a first-class parameter (`serial`/`asynchronous`/`thread_pool` task-processing models); serial traversal is constexpr-capable.
- Known TODO in-code: `erase_edge` disabled for `undirected_graph` with non-empty `EdgeMetaData` (DynamicGraph.hpp ~line 451).
- Graph is the backbone of much of sequoia: the CLI parser is a directed tree traversed depth-first, the test-dependency analyzer builds a directed graph of file dependencies, the suite machinery extracts a `directed_tree`, and `transition_checker` state-transition testing walks a directed graph.

**Maths/Geometry — Spaces.hpp** (HIGH LEVEL ONLY — churning): a strongly-typed vector/affine/convex-space framework, ~1800 lines with a ~200-line essay at the top stating the intent: sets underlying algebraic structures are type-level names; all computation happens in coordinates w.r.t. a basis, keeping "vector" rigorously distinct from "coordinate representation". Concept ladder ("weak" variants acknowledging floating-point's axiom failures) → `free_module` → `vector_space` → `convex_space`/`affine_space` (convex spaces generalize to half-lines/intervals via validators, e.g. `half_line_validator`). User-facing value type: `coordinates<Space, Basis, Origin, Validator>`; convenience `vec_coords` tagged by an `Arena` type so e.g. local-geometry vs texture coordinates are distinct types. `Algebra/Ratio.hpp` (2025) generalizes `std::ratio` to floating-point, feeding Physics unit conversions.

**Physics** (HIGH LEVEL ONLY — churning): dimensional analysis built *on top of* the spaces framework — its distinguishing idea vs e.g. mp-units: `physical_value<ValueSpace, Unit, Basis, Origin, Validator>` is coordinates in a typed space encoding the dimension. Per-unit validators express physical constraints (mass on a half-line; celsius affine with unit-defined origin); `composite_unit` + compile-time reduction machinery canonicalizes derived units; `inline namespace si` supplies the usual units and quantity aliases. Nothing else in sequoia consumes Physics yet.

**Sequences**: `linear_sequence` (on-demand arithmetic progression) and `monotonic_sequence` (container wrapper enforcing a Compare invariant on every mutation) — the latter underpins PartitionedData partition offsets, hence the entire graph stack. **Statistics**: iterator-based mean/variance/sd returning `std::optional`; sole customer is performance testing.
